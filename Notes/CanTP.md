#### CanTP（传输协议）：实现长于8字节的数据传输
    • 数据分段和重组
    • 流控机制
    • 支持功能寻址和物理寻址
    • 常用于UDS和某些需要传大量参数的场景
    • FF首帧
        ○ Byte0：0x10或0x11
        ○ Byte1：待传数据的总长度
        ○ Bytes 2~7：前6个字节的有效数据
    • FC流控帧
        ○ Byte 0 = 0x30
        ○ Byte 1：连发几个CF，0为无限发
        ○ Byte2：两个CF的间隔时间，0xF1=1ms（特殊）
        ○ Byte3~7：保留
        ○ 当规定的CF发完后，重发FC
    • CF连续帧
        ○ Byte 0 = 0x21 ~ 0x2F：最高两位 0010 表示是CF，低4位是序列号（SN）
        ○ Bytes 1~7：有效数据
    • SF单帧
